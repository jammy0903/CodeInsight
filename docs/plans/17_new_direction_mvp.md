# MVP 설계 계획서: CodeInsight

> **"AI가 대신 짜준 코드조차 이해하지 못하는 사람을, 스스로 설명할 수 있는 개발자로 만드는 앱"**

---

## 1. 프로젝트 정체성

### 1.1 한 줄 정의

```
코드를 써보면서, 눈으로 원리를 이해하는 프로그래밍 학습 앱
```

### 1.2 이 앱은 무엇이 아닌가 (중요)

| ❌ 아닌 것 | 이유 |
|-----------|------|
| 코딩테스트 플랫폼 | 로직/알고리즘 중심 |
| 문제은행 | 정답 맞추기 중심 |
| GPT 튜터 | 말로만 설명 |
| IDE | 결과만 보여줌 |
| 점수/랭킹 시스템 | 경쟁 유도 |

### 1.3 이 앱은 무엇인가

| ⭕ 요소 | 의미 |
|---------|------|
| 시뮬레이터 | 눈으로 이해 |
| 단계별 실행 | 한 줄씩 |
| 상태 변화 | 메모리/객체/값 시각화 |
| AI 해설자 | 코드 생성 ❌, 설명만 ⭕ |

---

## 2. 타겟 사용자

### 2.1 핵심 타겟 (2종)

#### ① 원래 타겟
> **문법·원리를 외워서 코딩하는 사람**

특징:
- 포인터, 참조, 객체 헷갈림
- "그냥 된다" 느낌으로 코딩
- 조금만 바뀌면 못 고침

#### ② 추가 타겟
> **AI로만 코딩해서 실제로는 이해 없이 복붙하는 사람**

특징:
- ChatGPT/Copilot 의존
- 코드가 왜 되는지 모름
- 에러 나면 또 AI에게 물어봄

### 2.2 공통 문제

| 증상 | 원인 |
|------|------|
| 코드가 왜 되는지 모름 | 실행 모델 없음 |
| 에러 나면 멈춤 | 내부 상태 상상 불가 |
| 조금만 바뀌면 못 고침 | 원리 이해 부족 |

## 3. 핵심 기능 (4개만)

### 3.1 실행 시뮬레이터 (심장) ⭐⭐⭐⭐⭐

```
코드 → 한 줄씩 실행 → 상태 변화 시각화
```

- 메모리 블록 (Stack/Heap)
- 변수 값 변화
- 포인터/참조 화살표
- 객체 생성/소멸

**지원 언어**: C → Python → Java (순차 확장)

#### 3.1.1 실제 메모리 레이아웃 시각화

실제 메모리 구조와 유사하게 시각화하여 학습 효과 극대화.

**C 언어 메모리 레이아웃**
```
┌─────────────────────────────────────┐
│           Memory Layout             │
├─────────────────────────────────────┤
│ High Address                0xFFFF  │
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐ │
│ │ STACK                      ↓    │ │  ← grows down
│ │ ┌───────────────────────────┐   │ │
│ │ │ main()                    │   │ │
│ │ │   a: 10        [0x7FFC]   │   │ │
│ │ │   p: 0x5000 ──────────────┼───┼─┤
│ │ └───────────────────────────┘   │ │
│ └─────────────────────────────────┘ │
│           ░░░ FREE ░░░              │
│ ┌─────────────────────────────────┐ │
│ │ HEAP                       ↑    │ │  ← grows up
│ │ │ 20 │           [0x5000] ◀─────┤ │
│ └─────────────────────────────────┘ │
│ ┌─────────────────────────────────┐ │
│ │ DATA (Initialized)              │ │
│ │   global_var: 100   [0x2000]    │ │
│ └─────────────────────────────────┘ │
│ ┌─────────────────────────────────┐ │
│ │ CODE (Text)                     │ │
│ │   main(), printf()...           │ │
│ └─────────────────────────────────┘ │
│ Low Address                 0x0000  │
└─────────────────────────────────────┘
```

#### 3.1.2 모듈형 시각화 컴포넌트 구조

언어별 확장이 쉬운 플러그인 형태의 구조:

```
visualizers/
├── core/                           # 공통 컴포넌트
│   ├── layout/
│   │   ├── MemoryContainer.tsx     # 전체 메모리 컨테이너
│   │   ├── Segment.tsx             # 메모리 세그먼트 틀
│   │   ├── SegmentHeader.tsx       # 세그먼트 헤더 (이름, 주소 범위)
│   │   ├── FreeSpace.tsx           # 빈 공간 표시
│   │   ├── AddressColumn.tsx       # 주소 표시 컬럼
│   │   └── GrowthArrow.tsx         # Stack↓ Heap↑ 화살표
│   ├── primitives/
│   │   ├── MemoryCell.tsx          # 단일 메모리 셀
│   │   ├── PointerArrow.tsx        # 포인터 화살표
│   │   └── HighlightBox.tsx        # 변경 하이라이트
│   └── registry.ts                 # 언어별 시각화 등록
│
├── c/                              # C 언어 전용
│   ├── CVisualizer.tsx             # C 메인 시각화
│   ├── config.ts                   # C 설정 (세그먼트, 색상)
│   └── segments/
│       ├── StackSegment.tsx        # Stack 세그먼트
│       ├── HeapSegment.tsx         # Heap 세그먼트
│       ├── DataSegment.tsx         # Data 세그먼트
│       └── CodeSegment.tsx         # Code 세그먼트
│
├── python/                         # Python 전용
│   └── (Phase 2)
│
└── java/                           # Java 전용
    └── (Phase 3)
```

**Registry 패턴**
```typescript
// visualizers/core/registry.ts
export const visualizerRegistry = {
  c: CVisualizer,
  python: PythonVisualizer,
  java: JavaVisualizer,
};

// 사용
const Visualizer = visualizerRegistry[language];
<Visualizer state={memoryState} />
```

### 3.2 AI 해설자 (자동 해설 + Q&A) ⭐⭐⭐⭐⭐

```
자동 해설: 줄 변경 시 → 짧은 설명 (1-2문장)
Q&A 대화: 질문 입력 → 상세한 답변
```

#### 3.2.1 API 분리

| 엔드포인트 | 용도 | 특징 |
|------------|------|------|
| `POST /api/ai/explain` | 자동 해설 | 짧게 (1-2문장), debounce 400ms |
| `POST /api/ai/chat` | Q&A 대화 | 상세하게, quizActive 체크 |

#### 3.2.2 컴포넌트 구조

```tsx
<DayPage>
  <AutoExplanation          // 자동 해설
    line={currentLine}
    code={code}
    debounceMs={400}
  />
  <ChatQA                   // Q&A 대화
    context={context}
    disabled={quizActive}   // 퀴즈 중 비활성화
  />
  {quizActive && <p>퀴즈를 먼저 풀어주세요</p>}
</DayPage>
```

#### 3.2.3 Props 전달 (Store 사용 안 함)

```tsx
// ✅ Props로 명시적 전달
<AIExplainer
  context={courseContext}      // 코스 정보
  currentLine={currentLine}    // 현재 줄 (시뮬레이터 연동)
  quizActive={!quizCompleted}  // 퀴즈 차단용
/>

// ❌ Store로 암묵적 공유 (사용 안 함)
```

#### ⭕ 할 수 있는 것
- 현재 줄의 동작 자동 설명
- 흔한 착각 포인트 집어줌
- 사용자 질문에 대한 답변 (Q&A)
- 비유/예시로 개념 설명

#### ❌ 하면 안 되는 것
- 코드 생성/수정
- 퀴즈 정답 직접 알려주기 (quizActive 시 Q&A 비활성화)

#### 예시 (자동 해설 - /api/ai/explain)
> "이 줄에서 p가 가리키는 a의 값을 20으로 바꿉니다."

#### 예시 (Q&A 대화 - /api/ai/chat)
> 👤 "근데 왜 *를 붙이면 가리키는 곳이 돼?"
> 🤖 "*는 '역참조 연산자'예요. p에 저장된 주소로 가서
>     그 위치의 값을 읽거나 쓰라는 뜻이에요."

### Phase 2: Python 확장

#### Python 메모리 레이아웃 시각화

Python은 "이름 → 객체" 참조 모델을 실제 메모리와 유사하게 시각화:

```
┌─────────────────────────────────────────────────────────┐
│                    Python Memory Model                   │
├─────────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────────┐ │
│ │ FRAMES (Call Stack)                            ↓    │ │
│ │ ┌─────────────────────────────────────────────────┐ │ │
│ │ │ Global Frame                                    │ │ │
│ │ │   a ─────────────────────────────────────┐      │ │ │
│ │ │   b ─────────────────────────────────────┤      │ │ │
│ │ │   x ────────────────────────────────┐    │      │ │ │
│ │ └─────────────────────────────────────┼────┼──────┘ │ │
│ │ ┌─────────────────────────────────────┼────┼──────┐ │ │
│ │ │ func() Frame                        │    │      │ │ │
│ │ │   local_var ────────────────────────┤    │      │ │ │
│ │ └─────────────────────────────────────┼────┼──────┘ │ │
│ └───────────────────────────────────────┼────┼────────┘ │
│                                         │    │          │
│                                         ▼    ▼          │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ OBJECTS (Heap)                                      │ │
│ │  ┌─────────────────────┐  ┌─────────────────────┐   │ │
│ │  │ list       mutable  │  │ int      immutable  │   │ │
│ │  │ ┌───┬───┬───┬───┐   │◀─│ 42                  │◀──│ │
│ │  │ │ 1 │ 2 │ 3 │ 4 │   │  └─────────────────────┘   │ │
│ │  │ └───┴───┴───┴───┘   │                            │ │
│ │  │ ⚠️ a, b 공유!       │                            │ │
│ │  └─────────────────────┘                            │ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

#### Java 메모리 레이아웃 시각화 (JVM 모델)

Java는 JVM 메모리 구조를 시각화:

```
┌─────────────────────────────────────────────────────────────┐
│                      JVM Memory Model                        │
├─────────────────────────────────────────────────────────────┤
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ STACK (Thread)                                     ↓    │ │
│ │ ┌─────────────────────────────────────────────────────┐ │ │
│ │ │ main()                                              │ │ │
│ │ │   person ──────────────────────────────────────┐    │ │ │
│ │ │   name ────────────────────────────────────────┤    │ │ │
│ │ │   count: 42  (primitive)                       │    │ │ │
│ │ └────────────────────────────────────────────────┼────┘ │ │
│ └──────────────────────────────────────────────────┼──────┘ │
│                                                    │        │
│                                                    ▼        │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ HEAP                                                    │ │
│ │  ┌─────────────────────────┐ ┌────────────────────────┐ │ │
│ │  │ Person                  │ │ String Pool            │ │ │
│ │  │ ┌─────────────────────┐ │ │ ┌────────────────────┐ │ │ │
│ │  │ │ name ───────────────┼─┼─┤▶│ "John" │ [intern] │ │ │ │
│ │  │ │ age: 25             │ │ │ └────────────────────┘ │ │ │
│ │  │ └─────────────────────┘ │ │                        │ │ │
│ │  │ 📦 id: 0x3A2F          │ └────────────────────────┘ │ │
│ │  └─────────────────────────┘                           │ │
│ │  ┌─────────────────────────┐                           │ │
│ │  │ int[] (length: 3)       │ ← 배열도 객체             │ │
│ │  │ ┌───┬───┬───┐          │                           │ │
│ │  │ │ 1 │ 2 │ 3 │          │                           │ │
│ │  │ └───┴───┴───┘          │                           │ │
│ │  └─────────────────────────┘                           │ │
│ └─────────────────────────────────────────────────────────┘ │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ METHOD AREA (Metaspace)                                 │ │
│ │   Person.class, String.class...                         │ │
│ │   static 변수들                                          │ │
│ └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

#### Java 커리큘럼 (5개)

| Day | 개념 | 착각 포인트 |
|-----|------|-------------|
| 1 | 참조 타입 | "변수가 객체다" ❌ → 변수는 참조, 객체는 Heap |
| 2 | == vs equals | "==로 비교하면 된다" ❌ → ==는 참조 비교 |
| 3 | String Pool | "new String()과 리터럴 같다" ❌ → 다른 객체 |

#### Java 특수 시각화 기능

1. **String Pool 하이라이트**: 같은 리터럴 → Pool에서 공유
2. **== vs equals 비교**: 두 참조가 같은 객체인지 시각적 표시
3. **GC 대상 표시**: 참조 없는 객체 회색 처리 (optional)
4. **static 영역 분리**: Method Area에 static 변수 표시
